Lightweight local LLM and text-to-speech workflow for privacy-sensitive, document-grounded question answering.

Project Overview

LLM + RAG + Text-to-Speech Assistant

Role: Undergraduate Researcher

As an undergraduate researcher, I developed this project to support local police departments with a practical, lightweight interface for asking operational questions over approved internal reference material. The workflow indexes documents through a retrieval-augmented generation (RAG) layer, retrieves the most relevant context, and provides grounded answers through a local LLM. Responses are then delivered in both text and speech using Piper TTS.

Implementation Notes

Design Approach

  • Combined local LLM reasoning with text-to-speech output for faster operator interaction.
  • Integrated a RAG retrieval flow to ground responses in indexed source documents.
  • Kept the architecture lightweight to run effectively on standard hardware.
  • Used Docker to containerize the runtime for repeatable setup and cross-platform consistency.

Technology Stack

Python Ollama Piper TTS RAG Workflow Docker Linux/Windows
Data Handling

Privacy and Repository Scope

Due to privacy and information sensitivity, only the text-to-speech and application-side integration system is public in the GitHub repository. No correlated operational documents, confidential source material, or agency-specific indexed data are included in the public codebase.
